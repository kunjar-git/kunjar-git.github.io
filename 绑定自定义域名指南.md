# 🌐 绑定自定义域名检查清单

## ✅ 已完成步骤

- [ ] 购买域名
- [ ] 配置 DNS 解析（A 记录 + CNAME）
- [ ] 在 GitHub Settings → Pages 中设置 Custom domain
- [ ] 启用 HTTPS
- [ ] 等待 DNS 生效

## 📝 需要修改的文件

如果域名变了，以下文件中的链接可能需要更新：

### 1. 社交链接（所有页面的 footer）

**文件**：
- `index.html`
- `about.html`
- `journal/index.html`
- `timeline.html`
- `posts/*.html`
- `tech.html`
- `coming-soon.html`

**修改内容**：
```html
<!-- 之前 -->
<a href="https://github.com/kunjar-git" target="_blank">GitHub</a>
<a href="mailto:your@email.com">Email</a>

<!-- 之后（如果需要） -->
<a href="https://github.com/kunjar-git" target="_blank">GitHub</a>
<a href="mailto:contact@yourdomain.com">Email</a>
```

### 2. SEO 相关（可选）

如果需要，可以在 `<head>` 中添加：

```html
<link rel="canonical" href="https://yourdomain.com/">
```

## 🐛 常见问题

### Q1: GitHub 显示 "DNS check failed"
**A**: 
- DNS 还没完全生效，耐心等待
- 检查 DNS 记录是否正确
- 使用 https://www.whatsmydns.net/ 检查全球传播情况

### Q2: 访问域名显示 404
**A**:
- 确认 GitHub Pages 中 Custom domain 已保存
- 确认仓库根目录有 `CNAME` 文件
- 检查 CNAME 文件内容是否正确（只有域名，无 https://）

### Q3: HTTPS 证书无效
**A**:
- 等待证书自动生成（通常 5-10 分钟）
- 清除浏览器缓存
- 如果超过 24 小时仍未生效，联系 GitHub Support

### Q4: 国内访问慢
**A**:
- GitHub Pages 在国内访问速度一般
- 可以考虑使用 Cloudflare CDN 加速
- 或者使用国内 CDN 服务（需要备案）

## 💡 小贴士

1. **域名选择**：
   - 个人博客推荐：`.com` / `.me` / `.blog`
   - 技术类推荐：`.dev` / `.io` / `.tech`
   - 国内用户：`.cn`（需要实名认证）

2. **隐私保护**：
   - 购买时开启 WHOIS 隐私保护（防止信息泄露）
   - 部分服务商免费提供，部分需要付费

3. **续费提醒**：
   - 设置到期提醒，避免域名过期
   - 建议一次性购买多年（通常有优惠）

4. **备案问题**：
   - 使用国内 CDN 需要 ICP 备案
   - 直接用 GitHub Pages 不需要备案（但访问可能不稳定）
