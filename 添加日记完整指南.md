## 方法 1: GitHub 在线编辑器（最简单！）⭐

**无需安装任何工具，浏览器即可完成！**

### 步骤 1: 打开 GitHub 仓库

访问：https://github.com/kunjar-git/kunjar-git.github.io

### 步骤 2: 创建日记 HTML 文件

1. 进入 `posts/` 文件夹
2. 点击 **Add file** → **Create new file**
3. 文件名格式：`YYYY-MM-DD-标题拼音.html`
   - 例如：`2026-03-26-today.html`
   - 或：`2026-03-26-my-day.html`

4. 复制以下模板，修改内容：

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>日记标题 - 个人日记</title>
    <link rel="stylesheet" href="../css/style.css">
    <link rel="stylesheet" href="../css/journal.css">
    <link href="https://fonts.googleapis.com/css2?family=Noto+Sans+SC:wght@300;400;500;700&display=swap" rel="stylesheet">
</head>
<body>
    <header class="header">
        <nav class="nav">
            <div class="nav-brand">
                <a href="../index.html">🏠 我的空间</a>
            </div>
            <ul class="nav-links">
                <li><a href="../index.html">首页</a></li>
                <li><a href="../journal/index.html">日记</a></li>
                <li><a href="../timeline.html">时间线</a></li>
                <li><a href="../about.html">关于我</a></li>
            </ul>
        </nav>
    </header>

    <main class="container">
        <button class="back-btn" onclick="history.back()">← 返回日记列表</button>

        <article class="post-content">
            <div class="post-header">
                <h1>日记标题</h1>
                <div class="post-meta-large">
                    <span>📅 2026 年 03 月 26 日</span>
                    <div class="post-tags">
                        <span class="post-tag thoughts">思考</span>
                        <span class="post-tag life">生活</span>
                    </div>
                </div>
            </div>
            <div class="post-body">
                <!-- 这里是日记正文，支持 HTML 标签 -->
                <h2>正文标题</h2>
                <p>这里是内容...</p>
                
                <h3>小标题</h3>
                <p>更多内容...</p>
                
                <ul>
                    <li>列表项 1</li>
                    <li>列表项 2</li>
                </ul>
                
                <blockquote>
                    <p>引用文字</p>
                </blockquote>
            </div>
        </article>
    </main>

    <footer class="footer">
        <div class="footer-content">
            <p>&copy; 2026 我的个人空间。Built with ❤️ on GitHub Pages</p>
            <div class="social-links">
                <a href="https://github.com/kunjar-git" target="_blank">GitHub</a>
                <a href="mailto:your@email.com">Email</a>
            </div>
        </div>
    </footer>
</body>
</html>
```

5. 滚动到页面底部
6. 填写提交信息：`Add: 新日记 - 日记标题`
7. 点击 **Commit changes**

### 步骤 3: 更新索引文件（简化版）

1. 在仓库页面，点击 `posts/` 文件夹
2. 找到 `posts-index.json`，点击它
3. 点击右上角的 **✏️ 编辑** 按钮

4. 在 `posts` 数组**最前面**添加：

```json
{
  "id": "2026-03-26-today",
  "title": "日记标题",
  "date": "2026-03-26",
  "tags": ["thoughts", "life"],
  "excerpt": "简短摘要，50 字以内，显示在列表中",
  "content": "关键词 1 关键词 2 关键词 3"
}
```

**注意**：
- ✅ `id` 要和文件名一致（去掉 `.html`）
- ✅ `tags` 可选：`life`, `thoughts`, `tech`, `travel`
- ✅ `content` 只需要写**关键词**（用于搜索），不需要完整内容！
- ✅ 确保 JSON 格式正确（逗号、引号）

5. 滚动到底部，填写：`Update: 添加新日记索引`
6. 点击 **Commit changes**

### 步骤 4: 更新时间线索引

1. 在 `posts/` 文件夹，找到 `timeline-data.json`，点击它
2. 点击右上角的 **✏️ 编辑** 按钮

3. 在 `timeline` 数组中添加：

```json
{
  "id": "2026-03-26-today",
  "date": "2026-03-26",
  "dateDisplay": "2026 年 03 月 26 日",
  "title": "日记标题",
  "description": "简短摘要，50 字以内",
  "tags": ["thoughts", "life"],
  "link": "posts/2026-03-26-today.html"
}
```

**注意**：
- ✅ `dateDisplay` 不需要星期几，只写日期
- ✅ `link` 路径要和文件名一致

4. 滚动到底部，填写：`Update: 添加时间线索引`
5. 点击 **Commit changes**

### 步骤 5: 等待部署

- GitHub Pages 会在 **1-2 分钟** 内自动部署
- 访问：https://kunjar-git.github.io/journal/index.html
- 按 `Ctrl + Shift + R` 强制刷新

**完成！** 🎉

## 🐛 常见问题

### Q1: 网站没更新？
**A:** 
1. 确认 `git push` 成功
2. 等 1-2 分钟（GitHub 构建需要时间）
3. 按 `Ctrl + Shift + R` 强制刷新

### Q2: 日记不显示？
**A:**
1. 检查 `posts-index.json` 是否更新
2. 检查 JSON 格式是否正确（用 https://jsonlint.com/ 验证）
3. 打开浏览器控制台（F12）看错误信息

### Q3: 时间线没更新？
**A:**
1. 检查 `timeline-data.json` 是否更新
2. 确保 `link` 字段指向正确的文件路径
3. 确保 `dateDisplay` 格式：`2026 年 03 月 26 日`（不要星期）

### Q4: 筛选功能不工作？
**A:**
1. 检查 `tags` 数组是否正确
2. 标签必须是：`life`, `thoughts`, `tech`, `travel`

### Q5: content 字段写什么？
**A:**
- 只需要写**关键词**，用于搜索
- 例如：`"content": "药神 电影 人性 思考 成长"`
- 不需要写完整的日记内容！

## 💡 小贴士

1. **先复制现有日记修改** - 比从头创建更简单
2. **用在线编辑器入门** - 熟悉后再用 Git
3. **每次只改一个文件** - 出错容易排查
4. **保存模板到本地** - 下次直接复制
5. **定期备份** - 防止误操作
6. **content 写关键词** - 不用写完整内容
7. **日期不要星期** - `2026 年 03 月 26 日` 即可
